好的，我来帮你梳理一下插件系统的架构： 当前项目中有两个主要插件类型： 1. MCP 插件 MCP（Model Context Protocol）插件主要负责与外部数据源和工具的通信。在 plugins.html 中可以看到配置了以下 MCP 服务： 
┌─────────────────────────────────────┐
│ MCP 服务列表                      │
├─────────────────────────────────────┤
│ context7-cli    ● 已连接    v0.3.0 │
│ web-search      ● 已连接    v1.2.1 │
│ local-fs        ○ 未连接    v2.0.0 │
│ github-api      ○ 未连接    v1.5.0 │
└─────────────────────────────────────┘
2. Skills 插件 Skills 是智能体的能力模块，可以从 GitHub 安装或本地导入： 
┌─────────────────────────────────────┐
│ Skills 列表                        │
├─────────────────────────────────────┤
│ agent-browser  ● 已安装    v1.1.0 │
│ ask-user       ● 已安装    v2.0.0 │
│ context7-cli   ● 已安装    v0.5.2 │
│ code-review    ○ 未安装    v1.0.0 │
└─────────────────────────────────────┘
架构总结：
MCP 是"连接器"层——管理与外部世界的通信通道；
Skills 是"能力"层——定义智能体可以执行的具体任务；
两者通过统一的插件面板管理，用户可以在 plugins.html 中切换标签查看和操作。
