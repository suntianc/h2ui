# Phase 07.1: Vue Preview Support - Context

**Gathered:** 2026-05-24
**Status:** Ready for planning

<domain>
## Phase Boundary

Add Vue SFC preview server support to the existing `h2ui preview` command. When output directory contains `.vue` files, automatically switch to Vue preview mode with `@vitejs/plugin-vue` instead of `@vitejs/plugin-react`. The preview server should detect file types and configure Vite accordingly.
</domain>

<decisions>
## Implementation Decisions

### Preview Server Framework Detection
- **D-01:** Auto-detect framework based on file types in output directory
- **D-02:** If `.vue` files are detected, automatically use `@vitejs/plugin-vue`
- **D-03:** If `.tsx/.jsx` files are detected, use `@vitejs/plugin-react`
- **D-04:** If mixed files (both Vue and React) are detected, output a warning and suggest separating into different output directories

### Vue Runtime Version
- **D-05:** Use Vue 3 (latest stable) — consistent with `--framework vue3` default behavior
- **D-06:** Use `<script setup>` syntax (Composition API) — consistent with Phase 7 Vue SFC output format

### Vue Preview App Bootstrapping
- **D-07:** Generate `main.ts` that directly mounts root component via `createApp().mount('#root')`
- **D-08:** Do NOT use App.vue wrapper — simpler structure matching React preview approach
- **D-09:** Use same `index.html` structure as React preview with `<title>h2ui Vue Preview</title>` for distinction

### File Watching for Vue
- **D-10:** Add `.vue` extension to watched file extensions (currently: `.tsx`, `.ts`, `.jsx`, `.js`, `.css`)
- **D-11:** On `.vue` file change: rebuild preview app only (no reload message via WebSocket)
- **D-12:** Use Vite's built-in HMR for Vue SFC — component-level updates without page refresh
- **D-13:** Keep recursive file watching enabled (`recursive: true`)

### React Preview Behavior (unchanged)
- **D-14:** React preview continues to use `location.reload()` via WebSocket
- **D-15:** WebSocket reload message is conditional — only send for React mode, not Vue (let Vite HMR handle it)

</decisions>
