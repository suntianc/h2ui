## What Was Built

### Task 1: Framework Detection and Vue Mode (dd681bf)

**`src/preview/server.ts`** extended with:
- `detectFramework(outputDir)` function - scans directory for `.vue` and `.tsx/.jsx` files, returns `'vue' | 'react' | 'mixed'`
- `Framework` and `FrameworkMode` types exported
- `PreviewServerOptions.framework` option (`'vue' | 'react' | 'auto'`, default `'auto'`)
- `syncOutputFiles` now copies `.vue` and `.ts` files
- `writePreviewApp` generates Vue-specific `main.ts` with `createApp().mount('#root')` pattern and `index.html` with title "h2ui Vue Preview"
- `buildPreviewApp` uses `vue()` plugin when framework is `'vue'`, `react()` otherwise
- `.vue` added to watched file extensions
- WebSocket reload only sent for React mode (Vue relies on Vite HMR)

**`package.json`** updated:
- Added `@vitejs/plugin-vue` as dev dependency

### Task 2: CLI Framework Flag (8f8db7f)

**`src/cli/commands/preview.ts`** extended with:
- `--framework <vue3|react|auto>` option (default: auto)
- Description updated to mention Vue and React support
- Framework info displayed at startup
- Maps `vue3` to `vue` for server API compatibility
