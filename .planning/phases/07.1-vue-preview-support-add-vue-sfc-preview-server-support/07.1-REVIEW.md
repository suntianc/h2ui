## Warnings

### WR-01: Silent JSON parse error swallowing in readHeadLinks

**File:** `src/preview/server.ts:95-100`
**Issue:** The `readHeadLinks` function catches errors silently when parsing `.h2ui-component-tree.json`, which can hide configuration errors.
```typescript
try {
  const tree = JSON.parse(fs.readFileSync(treePath, 'utf8'));
  if (tree && Array.isArray(tree.headLinks)) {
    return tree.headLinks;
  }
} catch (_) {}
```
**Fix:** Log a warning when the tree file exists but cannot be parsed, so operators know the headLinks are unavailable:
```typescript
} catch (_) {
  console.warn(`[preview] Warning: Failed to parse ${treePath}, headLinks will be empty`);
}
```

### WR-02: Reload message sent after failed rebuild

**File:** `src/preview/server.ts:255-273`
**Issue:** When a file change triggers a rebuild that fails, the WebSocket reload message is still sent to clients on line 268-272 (outside the try block). Clients will reload even though the new build did not complete, causing a broken or stale preview state.
```typescript
try {
  // ... rebuild logic ...
} catch (err: any) {
  console.warn(`[preview] Rebuild failed: ${err.message}`);
  return;  // return is here
}

// Only send WebSocket reload for React mode - Vue uses Vite HMR
if (detectedFramework === 'react') {
  wss.clients.forEach(...)  // This runs even if try block threw
}
```
**Fix:** Move the reload send inside the try block after a successful build, or introduce a success flag:
```typescript
let rebuildSuccess = false;
try {
  // ... rebuild logic ...
  rebuildSuccess = true;
} catch (err: any) {
  console.warn(`[preview] Rebuild failed: ${err.message}`);
  return;
}

if (rebuildSuccess && detectedFramework === 'react') {
  wss.clients.forEach((client) => {
    if (client.readyState === 1) {
      client.send(JSON.stringify({ type: 'reload', file: filename }));
    }
  });
}
```

### WR-03: Unhandled exception from JSON.parse in resolveRootComponentName

**File:** `src/preview/server.ts:54-57`
**Issue:** If `.h2ui-component-tree.json` exists but contains malformed JSON, `JSON.parse` will throw an unhandled exception, crashing the preview server.
```typescript
if (fs.existsSync(treePath)) {
  const tree = JSON.parse(fs.readFileSync(treePath, 'utf8')) as PreviewTreeNode;
  if (tree?.name) return tree.name;
}
```
**Fix:** Wrap the parse in a try-catch and fall through to file-based detection:
```typescript
if (fs.existsSync(treePath)) {
  try {
    const tree = JSON.parse(fs.readFileSync(treePath, 'utf8')) as PreviewTreeNode;
    if (tree?.name) return tree.name;
  } catch (_) {
    // Fall through to file-based detection
  }
}
```

### IN-01: Unused type import

**File:** `src/preview/server.ts:4`
**Issue:** `WebSocket` is imported as a type but never used as a value. The type annotation on line 236 (`ws: WebSocket`) uses it, but the import can be simplified.
```typescript
import { WebSocketServer, type WebSocket } from 'ws';
```
**Fix:** If only used for type annotation, the type-only import is correct. However, the variable `ws` on line 236 could be typed more narrowly using `WebSocket.WebSocket` from the `ws` package if needed. This is a minor style observation.
