  <read_first>

  <action>
    Extend `src/preview/server.ts` with Vue preview support:

    1. Add `detectFramework(outputDir: string): 'vue' | 'react' | 'mixed'` function:
       - Scan output directory for `.vue` files and `.tsx`/`.jsx` files
       - Return 'vue' if only .vue files exist
       - Return 'react' if only .tsx/.jsx files exist
       - Return 'mixed' if both exist (output warning)

    2. Extend `PreviewServerOptions` interface to include:
       - `framework?: 'vue' | 'react' | 'auto'` (default: 'auto')

    3. Modify `syncOutputFiles` to copy `.vue` files alongside existing extensions:
       - Current extensions: `.tsx`, `.jsx`, `.css`, `.module.css`
       - Add: `.vue`, `.ts` (for Vue main.ts)

    4. Modify `writePreviewApp` to support Vue mode:
       - For Vue: generate `index.html` with `<title>h2ui Vue Preview</title>` and `src/main.ts` (not .tsx)
       - For Vue: generate `main.ts` using `createApp().mount('#root')` — NO App.vue wrapper per D-07, D-08
       - Vue main.ts imports: `import RootComponent from './components/${rootComponentName}'`
       - Vue main.ts: `import { createApp } from 'vue'; createApp(RootComponent).mount('#root')`
       - For React: keep existing behavior with `main.tsx`

    5. Modify `buildPreviewApp` to use appropriate Vite plugin:
       - Import: `import vue from '@vitejs/plugin-vue'`
       - For Vue: `plugins: [vue()]` instead of `[react()]`
       - For React: keep `plugins: [react()]`

    6. Modify `startPreviewServer`:
       - If `options.framework === 'auto'`, call `detectFramework(outputDir)` to determine framework
       - If `detectFramework` returns 'mixed', output warning: `[preview] Warning: Mixed Vue and React files detected. Please use separate output directories.`
       - Pass detected framework to `writePreviewApp` and `buildPreviewApp`

    7. Add `.vue` to watched file extensions in the watcher:
       - Current: `.tsx`, `.ts`, `.jsx`, `.js`, `.css`
       - Add: `.vue`

    8. Conditional WebSocket reload behavior (per D-15):
       - For React: send `{ type: 'reload', file: filename }` to WebSocket clients (existing behavior)
       - For Vue: do NOT send reload message — let Vite HMR handle component updates
  </action>
  <acceptance_criteria>

    - Vue preview generates main.ts with createApp().mount() (not React createRoot)
    - Vue preview uses @vitejs/plugin-vue instead of @vitejs/plugin-react
    - .vue files are copied to components directory during syncOutputFiles
    - .vue extension is watched for file changes
    - Vue mode does NOT send WebSocket reload message (HMR handles it)
    - React mode continues to send WebSocket reload message
    - Vue preview index.html has title "h2ui Vue Preview"
  </acceptance_criteria>
