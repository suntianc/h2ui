# Plan 02: Enhanced h2ui Init ÔÇö Config Scaffold

## Objective

Extend `h2ui init` to generate a complete config scaffold with all available fields. The generated `.h2uirc` must be valid JSON that cosmiconfig can parse, with self-documenting `_comment` keys.

## Rationale

h2ui users need to discover available config options. The `init` command generates a scaffold with all fields, defaults, and inline documentation so users can edit it without external docs.

## must_haves

- `h2ui init` generates `.h2uirc` with all 5 config fields
- Generated file is valid JSON (parses with `JSON.parse`)
- cosmiconfig can load the generated file without errors
- Running `h2ui init` when file exists shows warning
- `--force` flag overwrites existing file
