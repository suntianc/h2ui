# Plan 01: Config File Loading with cosmiconfig

## Objective

Implement cosmiconfig-based config loading with proper search order (package.json → .h2uirc → .h2uirc.json → .config/h2uirc), JSON-only parsing, and CLI-flag merging. Missing config is not an error — falls through to defaults.

## Rationale

Foundation for Phase 3. Without config loading, no other config work is possible. This also completes CFG-01 (config file support) and CFG-02 (default output is TSX; configurable via flag or config file).

## must_haves

- CFG-01: User can configure via `.h2uirc` config file or `h2u` field in `package.json`
- CFG-02: Default output is TSX; configurable via flag or config file
- CLI flags override config file values
- Missing config file is not an error — falls through to defaults
