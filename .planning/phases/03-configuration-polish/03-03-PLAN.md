# Plan 03: CLI Spinner + Output Enhancement

## Objective

Integrate ora spinner during pipeline execution and enhance the final output display. Spinner runs for the full pipeline duration, then transitions to success/failure status with file count summary and component tree preview.

## Rationale

DX polish item from the roadmap. Spinner provides visual feedback during conversion (which can take seconds for large files). Enhanced output shows users exactly what was produced.

## must_haves

- Spinner animates during pipeline execution
- On success, spinner shows green ✔ with file count
- On error, spinner shows red ✖ with error message
- Component tree displays after spinner output
- Warnings display after spinner output
- Graceful degradation in non-TTY environments

<action>
Modify `src/cli/commands/convert.ts`:

1. Add import: `import ora from 'ora'`

2. After file validation and path resolution, before pipeline creation:
   ```typescript
   const spinner = ora('Parsing and converting HTML...').start();
   ```

3. After pipeline.run() completes, replace success/error reporting:

   **Success path (no errors, no strict mode failure):**
   ```typescript
   spinner.stop(); // Stop spinner before showing final output
   
   // Show warnings
   if (ctx.warnings.length > 0) {
     showWarningSummary(ctx.warnings);
   }
   
   // Show success with file count
   if (mergedConfig.split && ctx.componentTree) {
     const fileCount = ctx.components?.length ?? 1;
     console.log(`\n\x1b[32m✓\x1b[0m Wrote ${fileCount} files to ${mergedConfig.out}`);
     showComponentTree(ctx.componentTree);
   } else {
     showSuccess(ctx.outputPath!);
   }
   ```

   **Error path:**
   ```typescript
   spinner.stop();
   
   for (const err of ctx.errors) {
     showError(err);
   }
   
   // Show partial output even with errors
   if (ctx.outputPath) {
     showSuccess(ctx.outputPath);
   }
   
   process.exit(1);
   ```

   **Strict mode warning promotion:**
   ```typescript
   if (opts.strict && ctx.warnings.length > 0) {
     spinner.stop();
     showWarningSummary(ctx.warnings);
     showError('Strict mode: warnings promoted to errors');
     process.exit(1);
   }
   ```

4. Remove the old top-level `showSuccess` and `showError` calls in convertCommand (replaced by spinner-based reporting)
</action>

<acceptance_criteria>
- Spinner starts with text "Parsing and converting HTML..." when conversion begins
- Spinner animates during async pipeline execution
- On successful conversion with split mode: spinner stops, then shows "✓ Wrote N files to ./output/" followed by component tree
- On successful conversion without split: spinner stops, then shows "✓ Wrote: ./output/Component.tsx"
- On conversion error: spinner stops, then shows "✗ [error message]" with remaining errors listed
- In non-TTY environment (CI/pipe): spinner does not animate, plain text fallback
- TypeScript compilation passes
