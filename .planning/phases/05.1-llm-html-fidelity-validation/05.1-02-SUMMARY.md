# 05.1-02-SUMMARY: Unified LLM Fidelity Step

**Wave:** 2
**Completed:** 2026-05-22
**Requirements:** Fidelity-01, Fidelity-02

## Truths Confirmed
- LLM fidelity step validates original HTML vs generated components
- Pipeline step named 'llm-fidelity' replaces 'llm-review' + 'llm-modify'

## Changes Made

### New Files Created

#### src/llm/structured/fidelity.ts
- Created combined `FidelityResultSchema` with:
  - Review fields: `approved`, `boundary_changes`, `naming_suggestions`, `cleanup_hints`, `_fallback`
  - Modify fields: `components[]` with `name`, `code`, `rationale`
  - Fidelity fields: `fidelity_report` with `structure_match`, `attribute_preservation`, `text_content_match`, `css_preservation`, `fidelity_notes`
- Exports: `BoundaryChangeSchema`, `NamingSuggestionSchema`, `FidelityResultSchema`, `FidelityResult` type

#### src/llm/llm-fidelity.ts
- Created unified `runLLMFidelity()` service combining review, modify, and fidelity validation
- System prompt instructs LLM to:
  - REVIEW: Validate component boundaries, suggest naming improvements
  - MODIFY: Improve component code quality
  - FIDELITY: Validate that generated components match original HTML
- Passes original HTML to LLM prompt for fidelity comparison
- Re-exports `validateBeforeWrite` for pipeline step use

#### src/pipeline/steps/llm-fidelity.ts
- Created `llmFidelityStep` with `name: 'llm-fidelity'`
- Requires `ctx.html`, `ctx.componentTree`, AND `ctx.components` (new requirement for fidelity)
- Validates components with `validateBeforeWrite` before applying
- Graceful fallback on errors (D-11 behavior)

### Modified Files

#### src/types/pipeline.ts
- Updated `llmResult` type to include:
  - `components?: Array<{ name: string; code: string; rationale: string }>`
  - `fidelity_report?: { structure_match, attribute_preservation, text_content_match, css_preservation, fidelity_notes }`

#### src/cli/commands/convert.ts
- Replaced `llmReviewStep` + `llmModifyStep` imports with `llmFidelityStep`
- Added fidelity_report display in LLM results output

#### src/pipeline/steps/llm-review.ts
- Added `@deprecated` JSDoc noting use of `llmFidelityStep` instead

#### src/pipeline/steps/llm-modify.ts
- Added `@deprecated` JSDoc noting use of `llmFidelityStep` instead
