# Requirements Archive: v1.0

**Archived:** 2026-05-22
**Project:** h2ui
**Core Value:** Take any HTML page and produce production-ready React components with properly extracted styles — in one command.

---

## v1 Requirements — ALL COMPLETE

### CLI & Input/Output

- [x] **CLI-01**: User can run `h2ui <file>` with an HTML file path argument — Phase 1
- [x] **CLI-02**: Output component files are written to disk (default: `./output/`) — Phase 1
- [x] **CLI-03**: User can specify custom output directory via `--out` — Phase 1
- [x] **CLI-04**: User can get help text via `--help` — Phase 1
- [x] **CLI-05**: User can get version via `--version` — Phase 1
- [x] **CLI-06**: Tool shows clear error message for missing/invalid file paths — Phase 1

### HTML→JSX/TSX Conversion

- [x] **JSX-01**: `class` attributes are converted to `className` — Phase 1
- [x] **JSX-02**: `for` attributes are converted to `htmlFor` — Phase 1
- [x] **JSX-03**: HTML boolean attributes are converted to JSX boolean props — Phase 1
- [x] **JSX-04**: Inline `style` strings are converted to React style objects (camelCase) — Phase 1
- [x] **JSX-05**: Void elements (br, hr, img, input, etc.) use self-closing syntax — Phase 1
- [x] **JSX-06**: Elements without children use self-closing syntax — Phase 1
- [x] **JSX-07**: SVG attributes are converted to JSX camelCase — Phase 1
- [x] **JSX-08**: All HTML event handlers (`onclick` → `onClick`) are converted — Phase 1
- [x] **JSX-09**: `tabindex`, `maxlength`, and other hyphenated attrs are camelCased — Phase 1
- [x] **JSX-10**: Output is TypeScript `.tsx` by default — Phase 1
- [x] **JSX-11**: User can opt out of TypeScript to get plain `.jsx` via `--no-typescript` — Phase 1

### Component Splitting

- [x] **SPL-01**: Semantic HTML tags (`<header>`, `<nav>`, `<main>`, etc.) are detected as component boundaries — Phase 2
- [x] **SPL-02**: Each semantic section becomes its own component file — Phase 2
- [x] **SPL-03**: Parent components import child components — Phase 2
- [x] **SPL-04**: Repeated element patterns are extracted as reusable components — Phase 2
- [x] **SPL-05**: Component tree structure is shown in console output — Phase 2
- [x] **SPL-06**: Non-semantic `<div>` containers split when they have a distinct class/ID — Phase 4

### CSS Extraction (→ CSS Modules)

- [x] **CSS-01**: Inline `style` attributes are extracted to CSS Module files — Phase 2
- [x] **CSS-02**: Only explicitly-set CSS properties are included — Phase 2
- [x] **CSS-03**: Shared/common styles are deduplicated into shared CSS module — Phase 2
- [x] **CSS-04**: Inheritable styles inherit from parent element — Phase 2
- [x] **CSS-05**: CSS shorthand properties are condensed — Phase 2
- [x] **CSS-06**: `<style>` tags in HTML are extracted to separate CSS Module files — Phase 2
- [x] **CSS-07**: Components import their corresponding CSS Module files — Phase 2

### Configuration

- [x] **CFG-01**: User can configure via `.h2uirc` config file or `h2u` field in `package.json` — Phase 3
- [x] **CFG-02**: Default output is TSX; configurable via flag or config file — Phase 1

### LLM Integration

- [x] **LLM-01**: Optional LLM pass improves component naming — Phase 4
- [x] **LLM-02**: Configurable LLM provider (OpenAI, Anthropic, Ollama) — Phase 4
- [x] **LLM-03**: LLM suggests code cleanup / optimization improvements — Phase 4
- [x] **LLM-04**: Token estimation and cost warning before LLM calls — Phase 4
- [x] **LLM-05**: Caching of LLM results — **REJECTED** (D-06 decision: no caching)

### LLM Code Modification

- [x] **LLM-06**: LLM applies naming and cleanup changes directly to code — Phase 5
- [x] **LLM-07**: Interactive component tree preview in browser — Phase 5

### Polish & DX

- [x] **POL-01**: Browser preview server (optional live reload) — Phase 5

### CLI Simplification (Phase 05.1)

- [x] **CLI-07**: `--type tsx|jsx` replaces `--typescript`/`--no-typescript` — Phase 05.1
- [x] **CLI-08**: LLM enabled by default; `--llm off` to disable — Phase 05.1
- [x] **CLI-09**: Remove `--llm-mode`, `--llm-provider`, `--llm-model` from CLI — Phase 05.1

### Fidelity Validation (Phase 05.1)

- [x] **Fidelity-01**: LLM validates generated components against original HTML — Phase 05.1
- [x] **Fidelity-02**: Single unified LLM step (merge review+modify into fidelity) — Phase 05.1
